- [ ] **Step 2: Replace handleVolumeChange() with deduplication logic**

Find `function handleVolumeChange(level)` (around line 408) and replace the entire function plus the event listener (lines 408-423) with:

```javascript
var volumeDebounce = null;   // (already exists at line 111 — keep it)
var _volumeInFlight = false;
var _volumePending = null;

function handleVolumeChange(level) {
  // Instant visual feedback — no debounce for the label
  sidebarVolLabel.textContent = level + "%";

  clearTimeout(volumeDebounce);
  volumeDebounce = setTimeout(function () {
    _sendVolume(level);
  }, 100);
}

function _sendVolume(level) {
  if (_volumeInFlight) {
    _volumePending = level;
    return;
  }
  _volumeInFlight = true;
  _volumePending = null;
  apiJson("/api/config/volume", {
    method: "POST",
    body: JSON.stringify({ level: level })
  }).then(function () {
    _volumeInFlight = false;
    if (_volumePending !== null) {
      _sendVolume(_volumePending);
    }
  }).catch(function () {
    _volumeInFlight = false;
    if (_volumePending !== null) {
      _sendVolume(_volumePending);
    }
  });
}
```

Keep the event listener as-is:
```javascript
sidebarVolume.addEventListener("input", function () {
  handleVolumeChange(parseInt(sidebarVolume.value, 10));
});
```

Note: `_volumeInFlight` and `_volumePending` should be declared near the existing `volumeDebounce` at line 111, or locally within the IIFE scope where volume handling lives.

- [ ] **Step 2: Diagnose and fix the gap**

The issue is that on desktop the tab bar and content don't anchor to the top of `.main`. Likely fixes:

1. Ensure `.main` has no padding-top that creates a gap, or the first child (tab bar) has no margin-top
2. If `.tab-bar` has `flex: 1` (line 731), this makes it grow — it should NOT be `flex: 1`. The tab bar should be fixed height, not flexible. Change to `flex: 0 0 auto` or remove `flex` entirely.
3. Ensure `.tab-panel.active` flows immediately after `.tab-bar` with no gap

The most likely culprit is `.tab-bar` having `flex: 1` — this would cause it to expand and push content down on larger screens. Fix:

```css
.tab-bar {
  /* Remove flex: 1 — tab bar should not grow */
  display: flex;
  align-items: center;
  gap: 4px;
  height: 42px;
  /* flex: 1; ← REMOVE THIS */
  border-bottom: 1px solid var(--border);
  padding: 0 8px;
  overflow-x: auto;
}
```

If that's not the issue, investigate further. The goal is: tab bar immediately below header, content immediately below tab bar, no dead space on desktop.

```python
def _normalise_entry(entry):
    """Normalise an index.json entry to (file_path, label_or_None)."""
    if isinstance(entry, str):
        return entry, None
    if isinstance(entry, dict):
        return entry.get("file", ""), entry.get("label")
    return str(entry), None
```
