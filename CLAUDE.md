# BenderPi ‚Äî Claude Code Context

Full context for AI-assisted development on the BenderPi voice assistant project.

---

## What This Is

A Raspberry Pi 5 voice assistant with the personality of Bender Bending Rodriguez (Futurama). Responds to the wake word **"Hey Bender"**, engages in multi-turn conversation, controls Home Assistant devices, gives weather/news briefings, and falls back to the Claude API for unknown queries. Voice is generated by a fine-tuned Piper TTS model trained on Bender audio clips.

**GitHub:** `git@github.com:Schmalvis/benderpi.git`

---

## Design Philosophy

**Offline-first.** BenderPi is designed to run entirely locally ‚Äî internet access is not required for core functionality. The Hailo AI HAT+ (Hailo-10H, 40 TOPS NPU) provides dedicated hardware acceleration for STT (Whisper) and LLM (Qwen2.5-1.5B) inference on-device. Cloud APIs are a last resort, used only when local quality checks fail.

**AI response priority order:**
1. Static WAV clips ‚Äî instant, zero compute
2. Pre-generated TTS ‚Äî instant playback
3. Hailo NPU LLM ‚Äî Qwen2.5-1.5B-Instruct via HailoRT GenAI (on-chip, ~3-8s)
4. Ollama CPU fallback ‚Äî only if Hailo unavailable
5. Claude API (cloud) ‚Äî only if local quality check fails (hedge phrases, too short)

**Routing config (`bender_config.json`):**
- `ai_backend: "hybrid"` + `ai_routing.*: "local_first"` ‚Äî NPU first, cloud fallback on quality fail
- `ai_backend: "local_only"` ‚Äî never touch cloud
- `ai_backend: "cloud_only"` ‚Äî always use Claude API (avoid unless debugging)

## Hardware

| Component | Detail |
|---|---|
| Board | Raspberry Pi 5 |
| AI Accelerator | Hailo AI HAT+ (Hailo-10H, 40 TOPS NPU) ‚Äî LLM + STT + vision inference |
| Camera | Raspberry Pi AI Camera (IMX500) ‚Äî used for VLM / scene analysis |
| Audio HAT | Adafruit Voice Bonnet (WM8960 codec) ‚Äî I2S, 2x MEMS mics |
| Speaker | 3W passive, connected to Voice Bonnet |
| LEDs | 12x WS2812B addressable RGB |
| LED Data | GPIO 10 / SPI MOSI ‚Äî SPI enabled in `/boot/firmware/config.txt` |
| ALSA card | Card 2 (`seeed-2mic-voicecard`, `hw:2,0`) |

### Critical audio constraint
The WM8960 can only operate at **one sample rate** at a time. The mic (porcupine wake word) runs at 16000Hz; playback runs at 44100Hz. **You cannot have both the input and output streams open simultaneously** ‚Äî the output stream must be closed while listening for the wake word.

This is why `audio.py` uses `open_session()` / `close_session()` rather than a persistent stream.

## Conversation Architecture (converse mode)

```
wake word detected
    ‚Üí audio.open_session()         ‚Äî opens output stream (44100Hz)
    ‚Üí play greeting clip
    ‚Üí loop:
        STT (faster-whisper)       ‚Äî records until 1.5s silence, max 15s
        intent.classify(text)
        response priority chain:
            1. real_clip           ‚Äî original Bender WAV from speech/wav/
            2. pre_gen_tts         ‚Äî pre-built TTS from speech/responses/
            3. promoted_tts        ‚Äî promoted AI response as static WAV
            4. handler             ‚Äî weather/news/HA control
            5. ai_fallback         ‚Äî Claude API ‚Üí TTS
        audio.play(wav)
        conversation_log.log_turn()
        if DISMISSAL or 8s silence: break
    ‚Üí audio.close_session()        ‚Äî closes output stream (frees device for mic)
```

## Briefings (Weather + News)

`scripts/briefings.py` ‚Äî pre-generates WAV files, cached to `speech/responses/daily/`.

| Briefing | Source | TTL |
|---|---|---|
| Weather | HA `HA_WEATHER_ENTITY` state + forecast | 30 min |
| News | BBC RSS UK + England feeds (2 headlines each) | 2 hours |

Both refresh at service start (in a background thread) and lazily when TTL expires. Fallback WAV used if generation fails.

## Home Assistant Control

`scripts/handlers/ha_control.py` ‚Äî dynamic entity discovery, no static config needed.

- Fetches all `light.*`, `switch.*`, `climate.*` entities from HA REST API (cached 60s)
- Filters noise via `EXCLUDE_KEYWORDS` and `EXCLUDE_ENTITIES` sets
- Normalises friendly names (splits CamelCase, strips noise words: "light", "lamp", "led", etc.)
- Fuzzy-matches user's spoken room/device term against normalised names
- Climate: uses `climate.set_hvac_mode` (on=heat/off=off) and `climate.set_temperature`

**Known entity notes:**
- `climate.martins_office_radiator_trv_2` is the correct office TRV (exclude `_1`)
- `climate.0xc09b9efffe848bb4` is a duplicate TRV (excluded)
- Several Zigbee entities have hex IDs ‚Äî HA housekeeping task to rename them

## Conversation Logging

One JSON Lines file per day in `logs/YYYY-MM-DD.jsonl`. Each line is a `session_start`, `turn`, or `session_end` event.

**Turn methods:** `real_clip`, `pre_gen_tts`, `promoted_tts`, `handler_weather`, `handler_news`, `handler_ha`, `ai_fallback`, `error_fallback`

```bash
# Review logs ‚Äî shows method breakdown and AI promotion candidates
venv/bin/python scripts/review_log.py
venv/bin/python scripts/review_log.py --days 7
```

### Promoting AI responses to static WAVs

1. Run `review_log.py` ‚Äî it flags queries with ‚â•3 AI hits as `*** PROMOTE?`
2. Add entry to `PROMOTED_RESPONSES` in `prebuild_responses.py`
3. Run `venv/bin/python scripts/prebuild_responses.py`
4. Redeploy ‚Äî that query now plays a pre-built WAV instead of calling the API

## Audio Notes

- All WAVs: 44100Hz, mono, 16-bit (real clips natively, TTS resampled)
- `audio.open_session()` ‚Äî opens output stream, writes 100ms warm-up silence
- `audio.close_session()` ‚Äî writes 100ms tail silence, stops and closes stream
- `audio.play(wav)` ‚Äî 50ms pre-silence + clip + 200ms post-silence (prevents DAC pop)
- Single shared `pyaudio.PyAudio()` instance at module load (concurrent init crashes PortAudio)
- `audio.get_pa()` ‚Äî returns shared instance; use in `wait_for_wakeword()` instead of `pyaudio.PyAudio()`

## Auto-Deploy

Changes pushed to `main` are pulled automatically by a systemd timer on BenderPi.

```bash
# Check auto-pull status
sudo systemctl status bender-git-pull.timer
sudo journalctl -u bender-git-pull.service -n 20

# Manual pull
cd /home/pi/bender && git pull origin main
```

The timer polls every 5 minutes. If the pull changes any Python files or `.env.example`, it restarts `bender-converse` automatically.

## Session Handover

- **`HANDOVER.md`** ‚Äî committed context file for decisions, priorities, and notes between Claude sessions. **Update this before finishing any session that makes changes.**
- **`STATUS.md`** ‚Äî auto-generated on the Pi by `scripts/generate_status.py`. Contains performance metrics, health alerts, usage stats. **Read this at the start of a session for device status.** Gitignored ‚Äî regenerated after every deploy.
- **`scripts/generate_status.py`** ‚Äî generates STATUS.md from logs/metrics. Called by git_pull.sh after deploys. Can also run manually.
- **`scripts/watchdog.py`** ‚Äî health anomaly detection with configurable thresholds (`watchdog_config.json`). Feeds into STATUS.md.

## Web UI

A browser-based admin panel and puppet mode served by FastAPI. Frontend is **Svelte 4 + Tailwind CSS**, built with Vite.

### Running locally (development)
```bash
# Development (with hot reload)
cd web && npm run dev

# Production build
cd web && npm run build

# Serve (same as before)
BENDER_WEB_PIN=<your-pin> python -m uvicorn scripts.web.app:app --reload --port 8080
```

### Build and deploy
The Svelte app is built locally and `web/dist/` is committed to git.
No Node.js needed on the Pi ‚Äî auto-deploy via git pull serves the pre-built files.

### Service setup (on Pi)
```bash
sudo cp systemd/bender-web.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable bender-web
sudo systemctl start bender-web
```

### Sudoers (required for service actions)

## Adding / Changing Things

### Add a new intent response

1. Add text to the relevant list in `scripts/prebuild_responses.py`
2. Run `venv/bin/python scripts/prebuild_responses.py`
3. Restart service

### Add a new intent

1. Add `_PATTERNS` list and classify check in `scripts/intent.py`
2. Add handler branch in `scripts/wake_converse.py`
3. Test: `venv/bin/python scripts/intent.py`

### Change the AI model

In `.env`:
```
BENDER_AI_MODEL=claude-sonnet-4-6
```

### Force-refresh briefings

```bash
sudo systemctl restart bender-converse
```

## Known Issues / Quirks

- **webrtcvad** ‚Äî use `webrtcvad-wheels` (not `webrtcvad`) ‚Äî the original package's `pkg_resources` import fails on Python 3.13
- **lgpio / neopixel** ‚Äî must use `--system-site-packages` venv; not pip-installable
- **scipy** ‚Äî only in venv; always run prebuild via `venv/bin/python`
- **WM8960 sample rate conflict** ‚Äî output (44100Hz) and mic input (16000Hz) cannot be open simultaneously on `hw:2,0`
- **Piper output is 22050Hz** ‚Äî must be resampled before playback; `tts_generate.py` handles this
- **BBC Nottingham RSS** ‚Äî `feeds.bbci.co.uk/news/england/nottinghamshire/rss.xml` returns 404; use UK + England feeds instead
- **BBC RSS CDATA** ‚Äî `xml.etree.ElementTree` can't parse BBC's CDATA titles; use regex extraction
